# Module Boundaries

This refactor keeps the browser behavior stable while moving the old inline
JavaScript and CSS out of `index.html`.

The page loads:

- `src/styles/app.css` for the former inline CSS.
- `src/app/bootstrap.js` as the compatibility runtime bundle.

`src/app/bootstrap.js` is generated by concatenating the source modules below in
order. The modules still share one application closure, so this is a safe
transitional structure rather than full ES-module isolation.

## Runtime Source Order

1. `src/app/00-core-ui-state.js`
   App version, state, DOM references, accessibility labels, date/string helpers,
   configuration constants, Firebase app setup, and shared state containers.
2. `src/clinical/10-therapy-validation.js`
   Clinical record adapters, validation, medication autocomplete support,
   medication safety checks, FHIR export helpers, hashing, sync/conflict helpers,
   and clinical test exposure.
3. `src/parser/20-ohbp-parser.js`
   OHBP text parsing and patient-data normalization.
4. `src/parser/30-parser-tests.js`
   Parser regression tests, Ctrl+Alt+P capture, and parser-test privacy
   sanitization.
5. `src/ui/40-rendering-ui.js`
   Main UI rendering, patient dialogs, Firebase patient workflows, local draft
   storage UI, sync status, audit calls, and form hydration.
6. `src/print/50-print-layout.js`
   Temperature-list rendering, preview pages, page navigation, print layout, and
   print workflow.
7. `src/ui/60-speech-ui-and-events.js`
   Speech helpers, keyboard shortcuts, DOM event bindings, drag/drop import,
   startup flow, and final app initialization.

## Build Contract

After changing any module source, run:

```bash
npm run build:bootstrap
```

Before release or smoke testing, verify:

```bash
npm run check:bootstrap
npm run test:smoke
```

## Next Safe Extraction Steps

- Move Firebase patient storage and audit helpers out of `src/ui/40-rendering-ui.js`
  into `src/storage/`.
- Move FHIR export helpers from `src/clinical/10-therapy-validation.js` into
  `src/integration/`.
- Move general date/string/hash/id helpers from `src/app/00-core-ui-state.js`
  into `src/utils/`.
- Convert the compatibility bundle to native ES modules only after smoke tests
  cover all critical UI, Firebase, parser, and print workflows.
